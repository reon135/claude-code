1
Claude Code とは何か？
「指示するだけで実際に作ってくれるAI」

Claude.aiが「回答してくれるAI」なら、Claude Codeはパソコン上で実際にファイルを作り・編集し・実行してくれるAIです。
「TODOアプリを作って」と指示すると、コードを書くだけでなく、ファイルを生成して、テストを実行して、完成まで自律的にやり遂げてくれます。

💬 Claude.ai（通常チャット）
  • 📄 コードを表示してくれる
  • ❌ ファイルを作るのは自分
  • ❌ 実行・テストは自分
  • ❌ エラー修正は自分
⚡ Claude Code
  • 🗂️ ファイルを実際に作成・編集
  • ▶️ コードを実行してくれる
  • 🔍 テスト・デバッグも自動
  • 🔄 エラーが出たら自己修正
💡
プログラミング知識は不要！ 「ショッピングリストアプリを作って」「このデータをExcelにまとめて」「ウェブサイトのデザインを変えて」——自然な日本語で伝えるだけで、Claude Codeが自律的に実現します。コードを読む必要すらありません。

インストール（Mac 編）
デスクトップアプリとターミナル版、両方解説
方法① デスクトップアプリをインストール（初心者向け）

一番かんたんな方法です。Claude Desktopをインストールすれば、中にClaude Codeのタブが含まれています。

  1. claude.com/download を開いて「macOS」ボタンをクリック
  2. ダウンロードした Claude.dmg をダブルクリック
  3. Claudeのアイコンを「アプリケーション」フォルダにドラッグ＆ドロップ
  4. アプリを起動してClaudeアカウントでログイン（Pro以上）
  5. 上部タブから「Code」をクリックして完了
方法② ターミナルからインストール（上級・全機能利用可能）

すべての機能を使いたい場合はターミナル（CLI）版をインストールします。ターミナルはMacの「LaunchPad → ターミナル」から開けます。

ターミナル
# ── Step 1: Homebrewでインストール（推奨）──
$ brew install claude-code

# または公式インストーラーで直接インストール
$ curl -fsSL https://claude.ai/install.sh | sh

# ── Step 2: バージョン確認 ──
$ claude --version
Claude Code 2.x.x
💡
Homebrewとは？ Macで使えるアプリ管理ツールです。インストールされていない場合は brew.sh から1コマンドでインストールできます。Homebrewでのインストールは自動更新されません。定期的に brew upgrade claude-code を実行して最新版を維持しましょう。

インストール（Windows 編）
デスクトップアプリとWSL版を解説
方法① デスクトップアプリをインストール（初心者向け）
  1. claude.com/download を開いて「Windows」ボタンをクリック
  2. ダウンロードした ClaudeSetup.exe をダブルクリック
  3. インストールウィザードに従って進む（「はい」→「次へ」→「インストール」）
  4. スタートメニューからClaudeを起動してログイン
  5. 上部の「Code」タブをクリックして完了
方法② WSL経由でターミナル版をインストール（全機能利用）

WSL（Windows Subsystem for Linux）とは、Windows上でLinux（Mac風のターミナル環境）を動かす仕組みです。Claude Codeのフル機能を使うにはWSLが必要です。

PowerShell（管理者権限）
# ── Step 1: WSL2 をインストール（PowerShellで実行）──
PS> wsl --install
インストール中... PCを再起動してください

# ── Step 2: 再起動後、Ubuntuを起動して以下を実行 ──
$ curl -fsSL https://claude.ai/install.sh | sh

# ── Step 3: 確認 ──
$ claude --version
Claude Code 2.x.x
⚠️
WSLはWindows上のLinux環境です WSL 2はセキュリティのためのサンドボックスをサポートしています。WSL 1はサポートしていません。また、WindowsにはGit for Windowsが必要です。インストールされていない場合は先にインストールしてください。
方法③ WinGet でインストール（Windows 10/11）

Windows標準のパッケージマネージャー WinGet を使う方法です。PowerShellで実行できます。

PowerShell
PS> winget install Anthropic.ClaudeCode
インストール完了
📌
WinGetでのインストールは自動更新されません。定期的に winget upgrade Anthropic.ClaudeCode を実行して更新してください。

8
CLAUDE.md でClaudeに設定を記憶させる
毎回同じ説明をしなくて済む「AI専用メモ帳」

CLAUDE.mdはプロジェクトのルールや好みをClaudeに永続的に記憶させるファイルです。
「毎回コーディングルールを説明するのが面倒」「自分のスタイルで書いてほしい」という悩みを解決します。/init コマンドで自動生成できます。

📄 CLAUDE.md — 実際の設定例（コンテンツ販売ビジネス向け）
# プロジェクト：レオのコンテンツ販売自動化

## 基本情報
- ブランド名：レオン心理学
- 主な作業：Threadsの投稿文・note記事・LINEシナリオ作成
- ターゲット：SNS副業初心者・ソロ起業家

## ライティングルール
- 文体：です・ます調をベースに、口語を交える
- 【】で見出し、＜＞で強調（アスタリスク太字は使わない）
- 短い文と長い文を交互にリズムを作る
- 読者への語りかけを多用する（「〜ですよね？」等）

## 作業フォルダ構成
- posts/threads/ ← Threads投稿文
- posts/note/ ← note記事
- line/ ← LINEシナリオ

## 禁止事項
- マーケティング用語の乱用
- 横並びの「します・なります・です」の繰り返し
（上記が続いたら指摘してください）

🔧 よくある問題と解決策

問題1 「claude: command not found」と表示される
  • A ターミナルを再起動する：インストール直後は再起動が必要な場合があります。ターミナルを閉じて再度開いてください。
  • B Homebrewの場合：brew doctor を実行してエラーを確認。パスが通っていない場合は echo 'export PATH="/opt/homebrew/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc を実行。
問題2 Claudeが思ったように動かない・暴走する
  • A 即座にEscキーを押す：処理を中断できます。これが一番重要です。
  • B 指示をより具体的にする：「修正して」ではなく「〇〇ファイルの△△行目を□□に変更して」のように。
  • C 新しいセッションで再試行：/clear でリセットするか、ターミナルを再起動してクリーンな状態から始める。
問題3 使用制限に達してしまう
  • A 5時間待てばリセットされます。時間をおいて再試行してください。
  • B /compact を活用する：コンテキストを圧縮して使用量を節約できます。長いセッションの途中で定期的に実行しましょう。
  • C シンプルな質問は通常チャット（claude.ai）を使う：ファイル操作が不要なときはブラウザ版で対応すると節約できます。
